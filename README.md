
# Plan Jeju [ 🌴 제주도 여행계획 추천 서비스 ]

여행 스타일 설문을 기반으로 사용자에게 맞춤형 제주도 여행계획을 추천하고,  
여행계획 관리 및 여행후기 공유까지 가능한 웹 서비스입니다.

---

## 📌 프로젝트 개요

- 여행 스타일에 따른 맞춤형 제주도 여행계획 추천
- Quick / Full 설문 모드를 제공하여 사용자 선택권 강화
- 추천된 여행계획 저장 및 수정·삭제 기능 제공
- 여행 후기 공유 및 여행 스타일 기반 탐색 기능 제공
- 본 프로젝트는 프레임워크 없이 순수 JavaScript와 jQuery를 기반으로 구현되었습니다.

## ✨ 주요 기능

### 1️⃣ 여행 스타일 설문 기반 추천
- 사용자는 웹사이트에서 설문조사를 진행할 수 있습니다.
- 설문은 **Quick 모드 / Full 모드** 중 선택하여 진행할 수 있습니다.
- 설문 결과를 바탕으로 사용자 여행 스타일에 맞는 여행계획을 추천합니다.
- 추천된 여행계획은 로그인 후 저장할 수 있습니다.

#### ▪ Quick 모드
- 필수 정보만 입력하여 빠르게 설문 진행
- 빠른 추천 제공 (정확도 ▼)

#### ▪ Full 모드
- 상세한 설문 제공
- 여행 스타일에 더욱 적합한 여행계획 추천 (정확도 ▲)
<br>

### 2️⃣ 제주도 여행계획 작성 및 관리
- 사용자는 여행계획을 생성, 수정, 삭제할 수 있습니다.
- 여행 시작일 / 종료일
- 인원 수
- 목적지
- 예산
- 일차별 여행 일정 관리 가능
<br>

### 3️⃣ 여행 후기 작성 및 조회
- 사용자는 여행 후기를 이미지와 설명을 포함하여 작성할 수 있습니다.
- 여행 후기는 특정 여행계획과 연결할 수 있습니다.
- 여행 스타일별 필터링 기능 제공
- 게시물에 좋아요 기능 제공

## 🤝 Team Convention

### 🔹 커밋 규칙

본 프로젝트는 커밋 메시지를 통해 **작업 내용과 변경 이력을 명확히 파악**할 수 있도록 아래 규칙을 따릅니다.


### 📌 커밋 메시지 기본 형식

```text
[기능분류]: [작성내용] - [날짜]
```
- **기능분류:** 작업의 성격
- **수정/작성내용:** 추가·수정한 작업 내용을 간단히 요약
- **수정/작성일:** 작업을 완료한 날짜 (YYYY-MM-DD)

<br>

#### 기능분류(Type)
| 타입     | 의미         |
| ------ | ---------- |
| feat   | 새로운 기능 추가  |
| docs   | 문서 작성 / 수정 |
| fix    | 버그 수정      |
| refact | 코드 리팩토링    |
<br>

### Commit 규칙 상세
<br>

**📌 단일 파일 작성 시**
```text
[기능분류]: [작업내용] - [날짜]
```
- 작업 내용을 중심으로 간단히 작성

**예시**
```text
feat: 로그인 기능 추가 - 26.01.22
fix: 비밀번호 검증 로직 오류 수정 - 26.01.22
docs: README 설치 방법 보완 - 26.01.22
```

<br>

**📌 여러 파일 작성 시**
```text
[기능분류]: [파일명] - [작업내용] - [날짜]
```
- 작성내용 앞에 작업한 파일명 명시

**예시**
```text
feat: Login.html - 로그인 UI 추가 - 26.01.22
feat: authController.html - 로그인 API 구현 - 26.01.22
feat: Login.html, authController.html - 로그인 기능 구현 - 26.01.22
```

### 🔹 브랜치 전략

본 프로젝트는 아래와 같은 브랜치 구조를 사용합니다.

#### 📌 브랜치 구조

```text
main      (배포 브랜치)
 ↑
feature   (통합 브랜치)
 ↑
t1 ~ t7   (개인 작업 브랜치)
```

### 주의사항

1. 작업자는 개인에게 부여된 브랜치에서 작업을 수행해야한다.
2. 작업자는 팀장의 허락하에 통합 브랜치인 **feature**에 Merge한다.
